# Repo Upload Checklist

Use this checklist before creating or uploading the public GitHub repository.

## Included

- `README.md`
- Hub71 proof summary, build log, architecture, demo script, submission copy, and screenshot checklist.
- Six Hub71 proof screenshots in `screenshots/`.
- Screenshot explanation README in `screenshots/README.md`.
- Supporting public-safe docs in `docs/`.
- Suggested commit message in `COMMIT_MESSAGE.txt`.

## Safety Review

- No `.env` files included.
- No API keys, private tokens, passwords, or credentials included.
- No runtime JSON or local execution state included.
- No LaunchAgent files included.
- No n8n workflow exports included.
- No RUDRA source files included; RUDRA is referenced only as early proof.
- No GitHub push was performed by this packaging step.

## Claim Boundary

- Do not claim revenue, customers, partnerships, Hub71 acceptance, or a finished public SaaS.
- Keep OpenClaw, Hermes, and n8n described as partial unless newer proof changes their status.
- Keep SHAURYA Automation OS described as a working internal system being productized.

## Before Upload

- Review screenshots for private tabs, tokens, private URLs, or unrelated personal data.
- Confirm `README.md` matches the intended public positioning.
- Confirm `screenshots/` contains exactly six `.jpeg` proof screenshots.
- Upload only the files in this folder.
